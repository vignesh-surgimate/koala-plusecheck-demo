Action Log — every action, zero instrumentation

A custom reporter persists every Playwright API call, assertion and test step to logs/run-<ET-timestamp>.log. Page objects and specs need no logging code — the runner emits each action as a step and the reporter captures it. Real output:

[04:38:12.806] RUN START | env=staging | sandbox=ModMedSandbox | role=Coordinator | tests=5 | tz=America/New_York
[04:38:13.262] TEST   ▶ module-dataset.spec.ts › reads a LoginTest dataset by module + S.No
[04:38:13.297]   ASSERT Expect "toBeGreaterThan" (0ms)
[04:38:13.300]   ASSERT Expect "toContain" (1ms)
[04:38:13.307] TEST   ✓ PASSED — reads a LoginTest dataset by module + S.No (17ms)
[04:38:13.405] RUN END | status=passed | duration=678ms
UI runs additionally log:  ACTION page.goto(…)  ·  ACTION locator.click(…)  ·  STEP Navigate to dashboard

The Allure Workflow

npm run allure:clean          # fresh slate
npm run test:matrix:smoke     # all 20 legs append to the same allure-results
npm run report:allure         # ONE consolidated report across the whole customer base
npm run allure:serve          # one-shot temp report
Operational note: passing --reporter=… on the CLI overrides the configured reporters — summary.json, Allure and the action log only appear on runs without that flag.
