Overview

Framework Features — in depth

Every capability, each owned by one module with one responsibility, each verified by an automated gate.

Capability Catalog

🎛 Runner Manager

External run control: SQLite drives which tests run, in what priority order, with what descriptions and data-iteration counts. Collection-time interception means dropped tests are never even registered.

runner/ · DATASOURCE=db

🏥 Multi-tenant matrix

5 EMR sandboxes × 4 clinical roles = 20 legs from one command, discovered dynamically from the credentials backend — add a tenant's users and it joins the matrix with zero config.

npm run test:matrix

👥 Per-role concurrent sessions

Cross-role clinical workflows in one test: coordinatorPage schedules while surgeonPage reviews — isolated, pre-authenticated contexts.

fixtures/auth.fixture.ts

🧩 Layered DI fixtures

config → data → auth → page: tests declare what they need and receive it ready-made. No setup boilerplate, no hidden globals, interfaces at every seam.

fixtures/ (4 layers)

📄 POM + widget components

33 page classes (Login, Home, Dashboard, Agenda, Calendar + 11 Surgery tabs + 5 Patient tabs + 4 Admin + 3 General-Admin) compose 13 component wrappers (PrimeNG table, autocomplete chooser, modal, confirmation, tabs, batch-save overlay, toast, date picker, file upload, toolbar, navbar, quick-finder). Each handled correctly once, reused everywhere.

pages/ · page-components/

🗄 Module datasets

getDataset('LoginTest', 2).UserID — SQL rows → per-sandbox, per-environment JSON snapshots, plus output write-back for chained modules.

database/module-data/

🔐 Secure credential backend

Secrets never sit in JSON or env vars — the users table in SQLite is the single source of truth, consumed through the IUserSource interface.

database/user-sources/

🕓 ET timezone everywhere

Node process and every browser context pinned to America/New_York (DST-aware). Generated surgery dates and app-rendered dates can never disagree — proven from an IST machine.

config/timezone.ts

🛡 Flake mitigation system

Web-first assertions, component-owned waits, retry/safe-click/safe-fill, DOM-stability detection for PrimeNG re-renders — and lint rules that block sleeps and test conditionals.

utilities/stability.helper.ts

🖥 No-browser validation harnesses

The entire data/config layer (57 checks × 5 sandboxes) and the runner plan verify from the console in seconds — demoable in a meeting room, CI-gateable before any browser starts.

validate:data · validate:runner

📊 Six-channel reporting

HTML, Allure, summary.json, per-run action log, Slack digest, ELK analytics — one run feeds every audience from debugging engineer to leadership.

reporters/

📐 SOLID + generated docs

Principle-by-principle compliance mapping, JSDoc with examples on every class, 284 generated API pages, a tester's worked guide, and a refactor log with rationale for every change.

docs/ · npm run docs:api
