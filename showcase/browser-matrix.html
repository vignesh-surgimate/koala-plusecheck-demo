How browsers are wired — real config

// playwright.config.ts — shared options come from one base (config/playwright-base.ts)
const forEnv = (envName: string) => ({
  storageState: storageStatePath(role, client, envName),
  baseURL: getEnvironmentConfig(envName).baseUrl,
});

{
  name: 'staging-edge',
  dependencies: ['auth-setup'],
  use: { ...devices['Desktop Edge'], viewport: { width: 1366, height: 768 }, ...forEnv('staging') },
  grep: /@smoke/,
},

Run Commands

npx playwright test --project=staging-chrome     # primary full run
npx playwright test --project=staging-edge       # enterprise channel smoke
npx playwright test --project=staging-firefox --project=staging-webkit  # cross-engine
npm run test:matrix:smoke                        # × 20 sandbox/role legs
Why this split: the application officially targets desktop Chromium-class browsers; Firefox/WebKit run the regression tier to catch engine-specific CSS, focus and date-input differences early, while Edge smoke protects the channel most hospital IT departments standardize on.
